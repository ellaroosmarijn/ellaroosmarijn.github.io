Relative, Absolute and Fixed Positioning

Sprint Two

CSS Positioning.

The CSS position property defines the location, or position, of an element in a document. To determine its final position, one would define the offsetting distance from the left, right, top or bottom of a position property. The syntax for this is: position: relative;, position: absolute;, posiiton: fixed;.

When offsetting, a positive value will move the element inwards, and if a negative value is used it will offset the element outwards, as shown on the diagram to the right. However, it is important to note that without a defined position type, the directional properties will have no effect.

Positional properties provide a foundation for the directional properties to work with. The following are the five positional properties:

  1. Static (default),
  2. Realtive,
  3. Absolute,
  4. Fixed, and
  5. Sticky.

This explanation will only discuss relative, absolute and fixed positioning.

Relative Positioning

Elements with position: realtive; remain in the normal flow of the document, so that even once it has been repositioned other elemets around it continue to act as though it hasn't moved.

To move or offset the element, the left, right, top and bottom values are applied. In doing this, the developer will need to specify how many pixels (px) the element is to move. It will then move according to its original position.

For example, if we had a red box with the original position of

.red-box{ width: 50%; height: 40vh; background: red; position: relative; }

We can see here that the red box appears in the center of the sreen. It would remain in this position ven if we tried to move the element by a value of 0 left: 0; top: 0;, it remains at the same position. This is because moving by a value of 0 would no effect.

So what would happen if we modified the positioning so that the code looked like:

red-box{ width: 50%; height: 40vh; background: red; position: relative; left: 120px; top: 120px; }

Note: the only thing that has changed in the code is left: 120px; and top: 120px;.

Absolute Positioning

Elements with position: absolute; are removed from the normal flow of the document, so that other elements around it will act as though it is no longer there. Absolute elements are positioned according to the closest positioned parent (nearest containing element with an assigned position). It is important that the parent has been positioned using position: relative;. If there happens to be no parent, then the element will be posiitoned according to the initial containing block.

For example, if we set the red box to position: relative; and create a new blue box with positon: absolute;, so that the new code looks like:

.red-box{ display: flex; justify-content: center; align-items: center; width: 50%; height: 40vh; background: red; position: relative; }
.blue-box{ background: blue; width: 40%; height: 40%; position: absolute; }

Then the box positioning would look like the image below:

Here we can seee that the blue box has positioned itself with the centre of the parent (the red box). This is because no specific parametes have been set. Now, let's see what happens when we move the blue box by setting a parameter of top: 0;, so that the new code looked like:

.blue-box{ background: blue; width: 40%; height: 40%; position: absolute; top: 0; }

Let's see the changes in the image below:

Here we can see that the blue box has moved to the top of the parent element (the red box). This is because it has moved according to the parent element, so that a top value of 0 positions the blue box's top edge with a distance of 0px from the parent's top edge.

Fixed Positioning

Like elements with position: absolute;, elements with position: fixed; are also removed from the normal flow of the document. But unlike the absolute positioning, a fixed positon element is positioned relative to the html element (the browser window). This means that the element will always remain fixed at the specified location, regardless of a user scrolling the page. A common example of a fixed element is a header on a website that remains in place, even after scrolling further down the page. Let's created our own fixed element.

Begin by setting the blue box to position: fixed; and create a new green box with positon: relative; to see the effect of the fixed position while scrolling. The code should now look like:

.red-box{ width: 50%; height: 40vh; background: red; position: relative; }
.blue-box{ background: blue; width: 20%; height: 20%; position: fixed; top: 50px; right: 50px; }
.green-box{ width: 50%; height: 40vh; background: green; position: relative; }
